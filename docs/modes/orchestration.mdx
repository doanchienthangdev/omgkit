# Orchestration Mode

Coordinate multiple tasks and agents.

## Behavior
- Parallel task management
- Agent delegation
- Result aggregation
- Workflow optimization
- Progress tracking

## When to Use
- Complex features
- Multiple files
- Team coordination
- Sprint execution

## Pattern
```
┌─────────────────┐
│  Orchestrator   │
└────────┬────────┘
    ┌────┼────┐
    ▼    ▼    ▼
  [A]  [B]  [C]
    │    │    │
    └────┼────┘
         ▼
   [Aggregate]
```

## Settings
- Parallelism: Enabled
- Delegation: Automatic
- Sync points: Defined
- Aggregation: Required


## Activation

```bash
# Switch to this mode
/mode orchestration

# Check current mode
/mode
```

## Mode Behavior

When active, this mode affects:

- **Agent Behavior** - How agents approach tasks
- **Output Style** - Level of detail in responses
- **Decision Making** - Autonomy level

## Tips

<Note>
You can temporarily switch modes for specific commands, then switch back to your preferred mode.
</Note>
