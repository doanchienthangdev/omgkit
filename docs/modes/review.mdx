# Review Mode

Critical analysis for code review and quality.

## Behavior
- Find issues proactively
- Security-focused
- Performance analysis
- Constructive feedback
- Prioritize findings

## Checklist
- [ ] Security vulnerabilities
- [ ] Performance issues
- [ ] Code quality
- [ ] Best practices
- [ ] Edge cases

## Output Format
```markdown
## Review: [File/PR]

### Status: APPROVED | CHANGES_REQUESTED

### Critical Issues
1. [Issue] - [Location] - [Fix]

### Suggestions
1. [Improvement] - [Rationale]

### Positive Notes
- [Good practice observed]
```

## Settings
- Focus: Issues first
- Security: High priority
- Feedback: Constructive
- Severity: Prioritized


## Activation

```bash
# Switch to this mode
/mode review

# Check current mode
/mode
```

## Mode Behavior

When active, this mode affects:

- **Agent Behavior** - How agents approach tasks
- **Output Style** - Level of detail in responses
- **Decision Making** - Autonomy level
