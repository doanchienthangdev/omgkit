## Project Configuration

After running `omgkit init`, your project has a `.omgkit/config.yaml` file:

```yaml
# .omgkit/config.yaml
project:
  name: "your-project"
  type: "web"  # web, api, cli, library, mobile

# AI Model Routing
models:
  default: "claude-sonnet"
  planning: "claude-opus"
  implementation: "claude-sonnet"
  review: "claude-opus"

# Agent Settings
agents:
  enabled: true
  auto_handoff: true  # Agents pass work to each other

# Sprint Settings
sprint:
  default_duration: "2 weeks"
  task_size: "2-5 min"  # Target task granularity
  auto_propose: true    # AI proposes sprint tasks

# Mode Defaults
modes:
  default: "default"
  omega_threshold: "10x"  # Trigger omega thinking at 10x+

# Output
output:
  plans_dir: ".omgkit/plans"
  docs_dir: ".omgkit/docs"
  logs_dir: ".omgkit/logs"
```

### Model Routing

Route different tasks to different Claude models:

```yaml
models:
  # Use Opus for complex planning and review
  planning: "claude-opus"
  review: "claude-opus"

  # Use Sonnet for implementation (faster, cheaper)
  implementation: "claude-sonnet"

  # Use Haiku for simple tasks
  simple: "claude-haiku"
```

<Note>
  Model routing optimizes cost while maintaining quality where it matters most.
</Note>

### Agent Configuration

```yaml
agents:
  # Enable/disable the agent system
  enabled: true

  # Let agents hand off work to each other
  auto_handoff: true

  # Specific agent overrides
  overrides:
    planner:
      model: "claude-opus"  # Always use Opus for planning
    tester:
      coverage_target: 80   # Minimum coverage percentage
```

### Sprint Settings

```yaml
sprint:
  # Default sprint length
  default_duration: "2 weeks"

  # Target task size for breakdown
  task_size: "2-5 min"

  # AI automatically proposes tasks based on vision
  auto_propose: true

  # Maximum tasks per sprint
  max_tasks: 20

  # Require tests before marking complete
  require_tests: true
```

### Testing Configuration

Configure test enforcement for development commands in `.omgkit/workflow.yaml`:

```yaml
testing:
  # Enable/disable testing enforcement globally
  enabled: true

  # Enforcement level: soft, standard, strict
  enforcement:
    level: standard

  # Auto-generate test tasks for features
  auto_generate_tasks: true

  # Coverage requirements
  coverage_gates:
    unit:
      minimum: 80
      target: 90
    integration:
      minimum: 60
      target: 75
    overall:
      minimum: 75
      target: 85

  # Blocking behavior
  blocking:
    on_test_failure: true
    on_coverage_below_minimum: true
```

#### Enforcement Levels

| Level | Test Failure | Coverage Below Min | Missing Tests |
|-------|--------------|-------------------|---------------|
| `soft` | Warning | Warning | Warning |
| `standard` | Block | Block | Warning |
| `strict` | Block | Block | Block |

#### CLI Configuration

Manage testing configuration with the `omgkit config` command:

```bash
# View current testing settings
omgkit config list testing

# Set enforcement level
omgkit config set testing.enforcement.level strict

# Disable auto-generated test tasks
omgkit config set testing.auto_generate_tasks false

# Set coverage minimum
omgkit config set testing.coverage_gates.unit.minimum 85
```

#### Command-Level Overrides

Override testing settings per command:

```bash
# Skip test enforcement (soft mode only)
/dev:fix "bug" --no-test

# Force test enforcement
/dev:feature "login" --with-test

# Use strict enforcement
/sprint:team-run --test-level strict

# Set coverage target
/dev:tdd "auth" --coverage 95
```

```yaml
modes:

  token_efficient:

```

## Environment Variables

You can also configure OMGKIT via environment variables:

```bash
# Override default model
export OMGKIT_MODEL="claude-opus"

# Disable agent auto-handoff
export OMGKIT_AUTO_HANDOFF="false"

# Set default mode
export OMGKIT_MODE="omega"
```

## Settings File

The `.omgkit/settings.json` file controls permissions:

```json
{
  "permissions": {
    "file_write": true,
    "git_operations": true,
    "external_api": false,
    "shell_commands": true
  },
  "confirmations": {
    "before_commit": true,
    "before_deploy": true,
    "before_delete": true
  }
}
```

## Per-File Configuration

Add OMGKIT directives to specific files:

```typescript
// @omgkit-skip-review
// This file is auto-generated, skip code review

// @omgkit-owner planner
// The planner agent owns this file
```
