## Overview

OMGKIT includes a complete design system with 31+ curated themes that integrate seamlessly with shadcn/ui. The system supports both V1 (flat colors) and V2 (color scales, effects, animations) theme formats.

**Key Features:**
- 30+ curated themes across 5 categories
- V2 schema with 12-step color scales (Radix-compatible)
- Export to CSS, SCSS, Tailwind, Figma, and Style Dictionary
- Automatic color replacement during rebuild
- Full dark mode support

## Quick Start

<CodeGroup>
```bash Initialize with Theme
# Apply theme during init
omgkit init --theme neo-tokyo

# Or explore themes first
omgkit init --with-design
```

```bash Switch Theme Later
# List all themes
/design:themes

# Apply a theme
/design:theme neo-tokyo

# Preview current theme
/design:preview
```
</CodeGroup>

## Theme Versions

OMGKIT supports two theme schema versions:

| Feature | V1 | V2 |
|---------|----|----|
| Flat colors | ✓ | ✓ |
| 12-step color scales | - | ✓ |
| $ref token references | - | ✓ |
| Status colors (success/warning/info) | - | ✓ |
| Effects (glass, glow, gradient) | - | ✓ |
| Animations (keyframes) | - | ✓ |
| Alpha variants | - | ✓ |

### V1 Theme (Flat Colors)

Traditional theme format with direct HSL values:

```json
{
  "name": "Neo Tokyo",
  "id": "neo-tokyo",
  "category": "tech-ai",
  "colors": {
    "light": {
      "background": "0 0% 100%",
      "primary": "346.8 77.2% 49.8%",
      "primary-foreground": "355.7 100% 97.3%"
    },
    "dark": { ... }
  },
  "radius": "0.5rem"
}
```

### V2 Theme (Color Scales + Effects)

Advanced theme format with 12-step color scales, $ref references, and effects:

```json
{
  "version": "2.0",
  "name": "Electric Cyan V2",
  "id": "electric-cyan-v2",
  "category": "tech-ai",

  "scales": {
    "primary": {
      "name": "cyan",
      "steps": {
        "light": {
          "1": "185 60% 99%",
          "9": "186 91% 42.7%",
          "12": "186 95% 18%"
        },
        "dark": { ... }
      },
      "alpha": {
        "light": { "1": "186 91% 42.7% / 0.05", ... }
      }
    }
  },

  "semanticTokens": {
    "light": {
      "primary": { "$ref": "scales.primary.steps.light.9" },
      "primary-hover": { "$ref": "scales.primary.steps.light.10" }
    }
  },

  "statusColors": {
    "light": {
      "success": "151 55% 42%",
      "warning": "39 100% 50%",
      "info": "206 100% 50%"
    }
  },

  "effects": {
    "glassMorphism": { "backdropBlur": "12px" },
    "glow": { "default": "0 0 20px" }
  },

  "animations": {
    "shimmer": {
      "duration": "8s",
      "easing": "ease-in-out",
      "keyframes": { ... }
    }
  }
}
```

## How It Works

OMGKIT provides CSS variables that shadcn/ui components consume automatically.

```
Your Project
├── .omgkit/
│   └── design/
│       ├── theme.json    ← Theme configuration
│       ├── theme.css     ← CSS variables (:root + .dark)
│       └── backups/      ← Theme backup history
└── components/
    └── ui/               ← shadcn components use theme.css
```

### Generated CSS (V2)

```css
/* V2 themes generate extended variables */
@keyframes shimmer {
  0% { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}

@layer base {
  :root {
    /* 12-step color scales */
    --cyan-1: 185 60% 99%;
    --cyan-9: 186 91% 42.7%;
    --cyan-12: 186 95% 18%;

    /* Semantic tokens */
    --background: 0 0% 100%;
    --primary: 186 91% 42.7%;
    --primary-hover: 186 92% 38%;

    /* Status colors */
    --success: 151 55% 42%;
    --warning: 39 100% 50%;
    --info: 206 100% 50%;

    /* Effects */
    --glass-blur: 12px;
    --glow: 0 0 20px;

    /* Animations */
    --animation-shimmer: shimmer 8s ease-in-out infinite;

    /* Typography & spacing */
    --radius: 0.5rem;
    --font-sans: Inter, system-ui, sans-serif;
    --font-mono: JetBrains Mono, monospace;
  }

  .dark { /* dark mode overrides */ }
}
```

## Theme Export

Export your theme to various design tools and framework formats.

### Available Formats

| Format | Extension | Use Case |
|--------|-----------|----------|
| `css` | `.css` | Direct CSS variables |
| `scss` | `.scss` | Sass variables, maps, mixins |
| `tailwind` | `.js/.ts` | Tailwind CSS configuration |
| `figma` | `.json` | Figma design tokens |
| `style-dictionary` | `.json` | Style Dictionary tokens |

### Export Examples

<CodeGroup>
```bash Single Format
# Export CSS
/design:export css

# Export Figma tokens
/design:export figma --output ./tokens/

# Export Tailwind config
/design:export tailwind
```

```bash All Formats
# Export all formats at once
/design:export --all

# Export all to specific directory
/design:export --all --output ./exports/
```
</CodeGroup>

## Theme Rebuild

Change your project's entire design system with a single command.

### Rebuild with New Theme

<CodeGroup>
```bash CLI
omgkit design:rebuild neo-tokyo
```

```bash Slash Command
/design:rebuild neo-tokyo
```

```bash Preview Changes
# Dry run to see what will change
/design:rebuild neo-tokyo --dry
```
</CodeGroup>

### What Gets Updated

1. **Theme Configuration**
   - `.omgkit/design/theme.json` - Updated with new theme
   - `.omgkit/design/theme.css` - Regenerated CSS variables

2. **Tailwind Config**
   - `tailwind.config.ts` color palette extended

3. **Component Files**
   - Hardcoded colors replaced with theme variables

### V2 Theme Features in Rebuild

When using V2 themes, you get additional CSS variables:

```css
/* 12-step color scales */
--teal-1 through --teal-12
--teal-a1 through --teal-a12  /* Alpha variants */

/* Status colors */
--success, --warning, --info

/* Extended tokens */
--surface, --surface-hover
--primary-hover, --accent-hover
--panel, --overlay

/* Effects */
--glass-blur, --glow

/* Animations */
@keyframes shimmer { ... }
--animation-shimmer
```

### Color Mapping Reference

| Hardcoded | Theme Variable | Context |
|-----------|---------------|---------|
| `bg-white` | `bg-background` | Page background |
| `bg-gray-50` | `bg-muted` | Subtle background |
| `text-gray-900` | `text-foreground` | Primary text |
| `text-gray-600` | `text-muted-foreground` | Secondary text |
| `bg-blue-500` | `bg-primary` | Primary action |
| `bg-red-500` | `bg-destructive` | Danger action |
| `border-gray-200` | `border-border` | Standard border |

### Validation Checks

- Required fields (name, id, category)
- ID format (kebab-case)

- Color format (HSL values)

## shadcn/ui Integration

After applying a theme, add shadcn components that automatically use your colors:

```bash
# Add components
/design:add button card input

# Or manually
npx shadcn@latest add button
```

Components use CSS variables from your theme:

```tsx
// Button automatically uses --primary from theme.css
<Button>Click me</Button>

// V2 themes provide additional variants
<Button className="hover:bg-primary-hover">
  With hover state from V2 theme
</Button>

// Status colors
<Alert className="bg-success text-success-foreground">
  Success message
</Alert>
```
