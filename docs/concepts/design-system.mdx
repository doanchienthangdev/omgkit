## Overview

OMGKIT includes a complete design system with **30 curated themes** that integrate seamlessly with shadcn/ui. All themes use the **V2 schema** with 12-step color scales, effects, and animations.

**Key Features:**
- 30 curated themes across 5 categories (all V2 format)
- 12-step color scales (Radix-compatible)
- Alpha variants for transparency effects
- Status colors (success, warning, info)
- Effects (glassmorphism, glow, gradients)
- CSS animations (shimmer, pulse, fade)
- Export to CSS, SCSS, Tailwind, Figma, and Style Dictionary
- Automatic color replacement during rebuild
- Full dark mode support

## V2 Theme Schema

All 30 themes use the V2 schema format with these features:

| Feature | Description |
|---------|-------------|
| 12-step color scales | `--primary-1` through `--primary-12` |
| Alpha variants | `--primary-a1` through `--primary-a12` |
| $ref token references | Semantic tokens reference scale values |
| Status colors | `--success`, `--warning`, `--info`, `--destructive` |
| Effects | glassMorphism, glow, gradients |
| Animations | shimmer, pulse-glow, fade-in, slide-up |
| Backward compatibility | Includes flat `colors` block for V1 compatibility |

### Theme Structure

```json
{
  "version": "2.0",
  "name": "Neo Tokyo",
  "id": "neo-tokyo",
  "category": "tech-ai",
  "description": "Cyberpunk-inspired with neon rose...",

  "colorSystem": {
    "type": "radix",
    "version": "3.0"
  },

  "scales": {
    "primary": {
      "name": "rose",
      "hue": 347,
      "steps": {
        "light": {
          "1": "347 60% 99%",
          "9": "347 77% 50%",
          "12": "347 95% 18%"
        },
        "dark": { ... }
      },
      "alpha": {
        "light": { "1": "347 77% 50% / 0.05", ... }
      }
    },
    "neutral": { ... }
  },

  "semanticTokens": {
    "light": {
      "primary": { "$ref": "scales.primary.steps.light.9" },
      "primary-hover": { "$ref": "scales.primary.steps.light.10" },
      "surface": { "$ref": "scales.neutral.steps.light.2" }
    }
  },

  "statusColors": {
    "light": {
      "success": "151 55% 42%",
      "warning": "39 100% 50%",
      "info": "206 100% 50%"
    }
  },

  "effects": {
    "glassMorphism": { "backdropBlur": "12px" },
    "glow": { "default": "0 0 20px" },
    "gradient": { "primary": { "from": "...", "to": "..." } }
  },

  "animations": {
    "shimmer": { "duration": "8s", "easing": "ease-in-out" },
    "pulse-glow": { "duration": "2s" },
    "fade-in": { "duration": "0.3s" }
  },

  "colors": { ... },  // Backward compatible flat colors
  "radius": "0.5rem",
  "fontFamily": { ... }
}
```

## How It Works

OMGKIT provides CSS variables that shadcn/ui components consume automatically.

```
Your Project
├── .omgkit/
│   └── design/
│       ├── theme.json    ← Theme configuration (V2)
│       ├── theme.css     ← CSS variables (:root + .dark)
│       └── backups/      ← Theme backup history
└── components/
    └── ui/               ← shadcn components use theme.css
```

### Generated CSS

```css
/* All themes generate V2 extended variables */
@keyframes shimmer {
  0% { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}

@layer base {
  :root {
    /* 12-step color scales */
    --rose-1: 347 60% 99%;
    --rose-9: 347 77% 50%;
    --rose-12: 347 95% 18%;
    --rose-a5: 347 77% 50% / 0.3;

    /* Semantic tokens */
    --background: 0 0% 100%;
    --primary: 347 77% 50%;
    --primary-hover: 347 80% 45%;
    --surface: 347 30% 98%;

    /* Status colors */
    --success: 151 55% 42%;
    --warning: 39 100% 50%;
    --info: 206 100% 50%;

    /* Effects */
    --glass-blur: 12px;
    --glow: 0 0 20px;

    /* Animations */
    --animation-shimmer: shimmer 8s ease-in-out infinite;

    /* Typography & spacing */
    --radius: 0.5rem;
    --font-sans: Inter, system-ui, sans-serif;
  }

  .dark { /* dark mode overrides */ }
}
```

## Theme Export

Export your theme to various design tools and framework formats.

### Available Formats

| Format | Extension | Use Case |
|--------|-----------|----------|
| `css` | `.css` | Direct CSS variables |
| `scss` | `.scss` | Sass variables, maps, mixins |
| `tailwind` | `.js/.ts` | Tailwind CSS configuration |
| `figma` | `.json` | Figma design tokens |
| `style-dictionary` | `.json` | Style Dictionary tokens |

## Theme Rebuild

Change your project's entire design system with a single command.

### Rebuild with New Theme

<CodeGroup>
```bash CLI

```

```bash Slash Command
/design:rebuild neo-tokyo
```

```bash Preview Changes
# Dry run to see what will change
/design:rebuild neo-tokyo --dry
```

```bash Full Auto Mode
# Zero manual steps - scans ALL directories including tests
/design:rebuild neo-tokyo --full
```
</CodeGroup>

### Full Auto Mode (`--full`)

The `--full` flag enables a completely automatic rebuild with zero manual steps:

| Feature | Standard Mode | Full Mode |
|---------|--------------|-----------|
| Directory scanning | app/, components/, src/, pages/ | ALL directories (tests/, lib/, utils/, hooks/, etc.) |
| Color conversion | Basic mappings only | 200+ extended mappings with AI inference |
| Test files | Not scanned | Safely updates test files (skips assertions) |
| Warnings | Manual review needed | Zero warnings, all patterns auto-converted |

**Extended color mappings in Full Mode:**
- Green colors (green, emerald, teal) → `bg-success`
- Red colors (red, rose) → `bg-destructive`
- Yellow/Amber → `bg-warning`
- Cyan/Sky → `bg-info`
- Purple/Violet → `bg-accent`

```bash
# Full mode + dry run (preview all changes)
/design:rebuild neo-tokyo --full --dry
```

### What Gets Updated

1. **Theme Configuration**
   - `.omgkit/design/theme.json` - Updated with new theme
   - `.omgkit/design/theme.css` - Regenerated CSS variables

2. **Tailwind Config**
   - `tailwind.config.ts` color palette extended

3. **Component Files**
   - Hardcoded colors replaced with theme variables

### V2 Theme Features in Rebuild

All themes generate these CSS variables:

```css
/* 12-step color scales */
--rose-1 through --rose-12
--rose-a1 through --rose-a12  /* Alpha variants */

/* Status colors */
--success, --warning, --info

/* Extended tokens */
--surface, --surface-hover
--primary-hover, --accent-hover
--panel, --overlay

/* Effects */
--glass-blur, --glow

/* Animations */
@keyframes shimmer { ... }
--animation-shimmer
```

### Color Mapping Reference

| Hardcoded | Theme Variable | Context |
|-----------|---------------|---------|
| `bg-white` | `bg-background` | Page background |
| `bg-gray-50` | `bg-muted` | Subtle background |
| `text-gray-900` | `text-foreground` | Primary text |
| `text-gray-600` | `text-muted-foreground` | Secondary text |
| `bg-blue-500` | `bg-primary` | Primary action |
| `bg-red-500` | `bg-destructive` | Danger action |
| `border-gray-200` | `border-border` | Standard border |

### Validation Checks

- Required fields (name, id, category, version)
- Version must be "2.0"
- ID format (kebab-case)
- Valid category
- Color format (HSL values)
- $ref resolution
- Color scale structure (12 steps)
- Circular reference detection

## shadcn/ui Integration

After applying a theme, add shadcn components that automatically use your colors:

```bash
# Add components
/design:add button card input

# Or manually
npx shadcn@latest add button
```

Components use CSS variables from your theme:

```tsx
// Button automatically uses --primary from theme.css
<Button>Click me</Button>

// All themes provide hover states
<Button className="hover:bg-primary-hover">
  With hover state
</Button>

// Status colors available in all themes
<Alert className="bg-success text-success-foreground">
  Success message
</Alert>

// Use 12-step color scales
<div className="bg-rose-9 hover:bg-rose-10">
  Using color scale
</div>
```
