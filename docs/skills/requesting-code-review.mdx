# Requesting Code Review Skill

## Before Requesting
- [ ] Tests pass
- [ ] Self-reviewed
- [ ] No debug code
- [ ] Docs updated

## PR Description
```markdown
## Summary
[What changed and why]

## Changes
- [Specific change 1]
- [Specific change 2]

## Testing
- [ ] Unit tests added
- [ ] Manual testing done

## Screenshots
[If UI changes]

## Questions
[Specific areas needing review]
```

## Best Practices
- Keep PRs small (<400 lines)
- One concern per PR
- Descriptive commit messages
- Respond to feedback promptly
